คุณกำลังช่วยฉันในขั้นตอนที่ 1 ของ Connect-X Development Workflow: **อ่าน Notion Task**

ดูเอกสาร workflow ฉบับเต็มที่ C:\Users\USER\Documents\GitHub\CONNECT_X_DEV_WORKFLOW.md

ขั้นตอนการทำงาน:
1. ถามฉันว่า Notion task ID คืออะไร (เช่น CX-3534) และให้ link หรือ paste รายละเอียด task มา
2. อ่านและสรุปให้ฉัน:
   - Task title และ description
   - Acceptance criteria (เป็น checklist)
   - Repo / scope ที่ต้องแก้ (backend / frontend / ecosystem ไหนบ้าง)
   - Dependencies หรือ API contract (ถ้ามี)
   - Mockup / design (ถ้ามี)
3. ตั้งคำถามที่ยังไม่ชัดเจน เพื่อให้ฉันถาม PM ต่อ
4. เสนอชื่อ branch ตาม convention: `CX-XXXX-short-description`
5. บันทึกสรุปลงไฟล์ใน C:\Users\USER\Documents\GitHub\.cx-tasks\CX-XXXX-summary.md (สร้าง folder ถ้ายังไม่มี)

อย่าเพิ่งเขียน code — ขั้นนี้แค่ทำความเข้าใจและวางแผนเท่านั้น
